Architecture & Cost Overview — Scale-to-Zero GPU Kubernetes on Oracle Cloud
Prepared for Client Review
1

The Problem

Running GPU servers 24/7 is extremely expensive. A single NVIDIA A10 GPU instance costs ~$1,440/month if left running continuously. For AI workloads that only need GPUs when users are actively making requests, this is a massive waste.

Our solution: A Kubernetes cluster that keeps GPU servers turned off when nobody is using them, and automatically boots them on-demand when a request arrives.

2

Infrastructure Overview

The cluster runs on Oracle Cloud Infrastructure (OKE) and consists of two separate node pools:

🖥️
CPU Node Pool
Always Running — ~$27/month
  • Shape: VM.Standard.E4.Flex
  • 1 OCPU, 8 GB RAM
  • 1 node (fixed)

Runs:

  • KEDA HTTP Interceptor
  • Cluster Autoscaler
  • NVIDIA GPU Operator
🎮
GPU Node Pool
Scale-to-Zero — $2.00/hr only when active
  • Shape: VM.GPU.A10.1
  • 1× NVIDIA A10 (24 GB VRAM)
  • 0–2 nodes (dynamic)

Runs:

  • AI Model (Llama 3 8B)
  • GPU Dashboard

CPU Node acts as the "receptionist" — it receives all web requests, decides if a GPU is needed, and manages the scaling process.

GPU Node contains the NVIDIA A10 GPU. It only exists when there are active requests. After 5 minutes idle, it is automatically shut down.

3

How a Request Flows Through the System

ScenarioResponse Time
Cold start (GPU is off)~5–10 minutes (first request only)
Warm (GPU already running)Instant (milliseconds)
Scale down (no activity)5 min idle → GPU shuts off
Note: The cold start wait only happens for the very first request after idle. Once the GPU is running, all subsequent requests are instant.
